# 证券池调库申请需求说明

## 目标与角色

研究员或业务人员检索证券，查看当前池状态，在权限范围内发起调入或调出申请，并在提交前完成关系、权限和风控校验。

## 核心业务线

1. 按代码、名称、类型分页检索证券并查看详情。
2. 查询当前用户可操作的调入池和调出池。
3. 查看证券和发行主体当前所在投资池状态。
4. 选择目标池后执行提交前校验，返回可读的失败原因。
5. 提交手工、批量或 Excel 来源的调整申请。
6. 按联动、互斥、关联规则生成同批次从属记录。
7. 查看证券历史调整记录和对应流程步骤。

## 接口清单

| 业务线 | 接口 |
|---|---|
| 证券上下文 | `/api/v1/securityPoolAdjust/querySecurityPage`、`querySecurityDetail` |
| 池范围与状态 | `queryAdjustPoolList`、`querySecurityPoolStatus` |
| 校验与提交 | `checkAdjust`、`addAdjustLog` |
| 追踪查询 | `queryAdjustLogList`、`queryAdjustStepList` |

## 关键校验

- 证券、目标池和调整方向必填，目标池必须在用户可调整权限范围内。
- 调入已在池证券、调出不在池证券、重复待审批申请应被拦截。
- 同批联动和互斥记录共享主流程，不重复创建审批步骤。
- 未最终通过的申请不得生成有效池状态。

## 验收标准

- 校验失败时不生成调整日志和流程步骤。
- 提交成功后主记录、从属记录、批次号和初始步骤一致。
- `SecurityPoolAdjustApiTest` 覆盖查询、校验、提交和追踪业务线。
